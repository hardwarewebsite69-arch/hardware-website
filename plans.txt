# Final architecture

Build the site as a **lean catalog + quote system**, not a full ecommerce store.

Replace all branding placeholders with "Amroz Traders" and all contact numbers with "0713355507" across UI, database seed, and settings defaults.

The priority is:

1. Show products clearly
2. Let users search fast
3. Let users request a quote fast
4. Let staff manage products and quotes easily

Anything beyond that is later.

## 4) Admin scope

Keep admin small.

```text
Admin
├── Dashboard
├── Products
├── Categories
├── Quotes
├── Media
└── Settings
```

Do not add orders, promotions, brands, or inventory modules yet unless the business actually needs them.

## 6) Cloudinary usage

Use Cloudinary only for:

* product images
* quote file uploads
* image resizing
* thumbnails
* compression

Store only the Cloudinary URL, public ID, and metadata in Supabase.

---

## 7) Quote system flow

### Upload Materials List

For PDF, image, Excel, or document uploads.

### Manual Item Entry

For customers who already know what they need.

This is the correct MVP because it covers most real hardware buying behavior without forcing full ecommerce complexity.
